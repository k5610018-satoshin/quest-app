# 5年4組 観点別評価記録アプリ

戸田小学校 5年4組（佐藤先生）専用の観点別評価入力ツール。
教科×単元×観点を選び、児童名簿28名の各児童ボタンに並ぶ [A][B][C] のミニボタンをタップするだけで評価が記録される。

`C:\Users\K5610\interaction-app\` の交友関係記録アプリと同じデザインシステム・操作感を踏襲。データは完全分離。

```
C:\Users\K5610\evaluation-app\index.html を ブラウザで開くだけ
```

サーバ不要。Chrome / Edge どちらでもOK。データは localStorage に保存される。

## 6つのタブ

| タブ | 用途 |
|------|------|
| **記録** | 教科・単元・観点を選び、児童ボタンの[A][B][C]をタップして1タップ評価 |
| **集計** | 教科ごとに児童×単元×観点のクロス表 / CSV出力 |
| **分布** | 単元×観点ごとのABC分布をバーチャート表示 + 要配慮児童の状況 |
| **履歴** | 全記録一覧（最大500件表示） / フィルタ / 複数選択削除 |
| **単元管理** | 教科ごとの単元一覧 / 新規追加 / 編集 / 複製 / アーカイブ |
| **設定** | JSON/CSVエクスポート・インポート / AI副担任向け出力 / 統計 / リセット |

---

## キーボードショートカット

| キー | 動作 |
|------|------|
| `1`〜`28` | 出席番号で児童を選択（1,2は350ms待ち、3〜9は即確定、2桁で確定） |
| `A` / `B` / `C` | 選択中の児童に評価を入力（観点軸モード） |
| `Space` | 次の未評価児童へフォーカス移動（観点軸モード） |
| `F1` / `F2` / `F3` | 観点切替（知識・思考・態度） |
| `Tab` | モード切替（観点軸 ↔ 児童軸） |
| `Esc` | 選択クリア / モーダルを閉じる |
| `Ctrl`+`Z` | 直前の評価を取り消し（一括操作も対応） |
| `Shift`+`?` | ヘルプ表示 |

## 操作の流れ（典型例）

### A. 1単元 × 1観点で全員評価（最速）

1. **記録タブ** で教科を選ぶ（例: 国語）
2. 単元を選ぶ（例: 銀河）
3. モード = **観点軸**
4. 観点切替タブで観点を選ぶ（例: 思考・判断・表現）
5. 評価基準カードで規準を確認
6. 児童ボタンの [A][B][C] をタップして評価
   - 同じ評価を再タップで取消
   - 別の評価をタップで上書き（古い記録は履歴として残る）
7. 進捗カウンターで「12/28人 A:5 B:6 C:1」を確認
8. 全員評価できたら、観点切替で「知識」「態度」も同様に

### B. 1児童について3観点まとめて評価

1. モード = **児童軸**
2. 児童カードに3観点（知/思/態）が縦に並ぶ
3. 児童名をクリックすると評価基準つきポップオーバーが開く（深掘り入力）

### C. 全員一括A評価（誤操作防止）

- 「全員A」ボタン → 確認ダイアログ → 一括設定（個別Undo可能）
- `Ctrl+Z` で取消可能（toastで案内）

## データ構造

### レコード形式（localStorage）

```js
{
  id: 'r-...',
  timestamp: '2026-04-30T10:15:00.000Z',
  date: '2026-04-30',
  subject: 'kokugo',           // 教科ID
  unitId: 'r-uuid',            // 単元ID
  viewpoint: 'thinking',       // 観点ID (knowledge/thinking/attitude)
  studentId: 5,
  grade: 'A',                  // A/B/C
  superseded: false            // true=上書き履歴として残ったもの
}
```

**重複ガード**: 同じ `subject/unitId/viewpoint/studentId` に対して新規記録すると、古い記録は `superseded: true` フラグが立つ（削除はしない）。集計時は最新1件（`superseded=false` && 最新タイムスタンプ）のみ採用。

### 単元データ

```js
{
  id: 'r-uuid',
  subject: 'kokugo',
  name: '銀河',
  criteria: {
    knowledge: '物語の場面の様子を表す言葉を理解している',
    thinking:  '登場人物の心情を場面の様子と関連付けて読んでいる',
    attitude:  '進んで読み、感想を交流しようとしている'
  },
  created_at: '...',
  archived: false
}
```

### localStorage キー

- `evaluationApp_v1` … メインデータ
- `evaluationApp_v1_backup` … 1世代前の自動退避
- `evaluationApp_lastBackup` … 最終手動エクスポート時刻

interaction-app（`interactionApp_v1`）とは完全に別キー。

## 運用のコツ

- **1単元1観点で全員一気に評価**するのが速い。観点軸モードで観点を切替えながら使う
- 同じ児童・観点を再評価すると古い記録は**履歴として残り**、最新が有効になる
- 児童ボタン左下の**緑●**=今日評価済み、**赤い破線枠**=要配慮児童で未評価
- マーク**●**=要配慮児童（10名）、**■**=観察優先児童（4名）
- 週1回は **設定タブ→JSONエクスポート** でバックアップ推奨（ヘルスバッジで日数確認可）
- リセットには直近24時間以内のエクスポート + `DELETE` 入力が必要

## 動作確認手順

1. ブラウザで `index.html` を開く
2. 記録タブで教科「国語」、単元「単元未設定」（初期作成済み）が選ばれていることを確認
3. 児童ボタンの [A] をクリック → 緑色になり、進捗が「1/28 A:1」になる
4. 同じ [A] を再クリック → 取消される
5. [B] をクリック → 青色に変わる
6. 観点切替タブで「思考・判断・表現」に変更 → グリッドがリセット表示される
7. **単元管理タブ** → 「＋ 新しい単元を追加」 → 教科:国語、単元名:銀河、3観点の評価基準を入力 → 保存
8. **記録タブ** に戻ると、単元プルダウンに「銀河」が現れている
9. 「銀河」を選んで評価をいくつか入力
10. **集計タブ** に切替 → 教科:国語 → 児童×単元×観点のクロス表が表示される
11. **分布タブ** に切替 → ABC分布バーが描画される、要配慮児童の状況が下部に出る
12. **設定タブ → 📥 JSONエクスポート** → `evaluation-YYYY-MM-DD.json` がダウンロードされる
13. **設定タブ → 🤖 AI副担任向け出力** → `evaluation-fukutannin-YYYY-MM-DD.json` がダウンロードされる
