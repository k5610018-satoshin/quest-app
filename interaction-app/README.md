# 5年4組 交友関係記録アプリ

休み時間・授業時間に「誰と誰が一緒に過ごしているか」を1タップで記録するPC用Webアプリ。

## 起動方法

`index.html` をダブルクリックするだけ。ブラウザで開きます。

ローカルサーバーで動かす場合（複数PC共有等）:
```bash
cd C:\Users\K5610\interaction-app
python -m http.server 8000
# → http://localhost:8000 にアクセス
```

ホーム画面ショートカット作成（おすすめ）:
1. ブラウザで `index.html` を開く
2. ブラウザのアドレスバー隣の「インストール」アイコンをクリック
3. デスクトップにアプリとして追加される

## 基本操作（記録の流れ）

1. **シーン選択** — 上部から「業間休み」「昼休み」「授業中」など1クリック
2. **モード選択** — シンプル(相手のみ) / 相手×活動 を切替
3. **主役児童をクリック** — 名簿グリッドから観察した子をタップ
4. **一緒にいた子をクリック** — 複数選択可（タップで切替）
5. **「保存して次へ」** — 主役だけクリアされ、すぐ次の児童を選べる

### ショートカット

- `↶ 取り消し` — 直前の記録を取り消し
- `↻ 同じ組合せで続ける` — 直前メンバーを残して次の主役を選ぶ
- 特殊状態ボタン — `🚶 一人で` `👨‍🏫 先生と` `🌐 他クラスと`

### 双方向反映の仕組み

データ保存は主役視点で1件のみ（A→[B,C]）。集計時に展開され：
- B視点でも「A、C と一緒」として表示
- C視点でも「A、B と一緒」として表示
- ペア共起もすべて自動カウント

## 機能タブ

| タブ | 内容 |
|------|------|
| **記録** | メイン入力画面 + 直近記録 + 観察してない子TOP3 |
| **集計** | 児童別の相手TOP8をシーン/カテゴリ/期間でフィルタ |
| **比較** | 休み時間 vs 授業時間 のJaccard類似度（同じ相手と関わるか） |
| **ソシオグラム** | クラス全体の関係図（円形配置・線太さ＝共起回数） |
| **履歴** | 全記録の一覧・選択削除 |
| **設定** | エクスポート/インポート・シーン名編集・活動リスト編集 |

## データ保存

- ブラウザの `localStorage` に保存（端末・ブラウザに依存）
- **必ず週1回程度はJSONエクスポートでバックアップ**
- 別PC・別ブラウザで使う場合はエクスポート→インポートで移行

## 児童マーク

- ★ 黄: 要配慮児童（長田優真・愛玲菜・伊藤煌・橋本彩希・下釜有真・斉藤初実・井上笑菜・服部実波・山崎玲央）
- 👁 紫: 観察優先（4日間で名前が出ていなかった4名）

これらは `students.js` の `highlight` / `watch` フラグで制御。後で更新可。

## ファイル構成

```
interaction-app/

├── styles.css      ← スタイル (PC前提)

```

## 名簿の更新

転入・転出があった場合は `students.js` を編集（同時に `students.json` も同期推奨）。
編集後、ブラウザをリロードで反映。

## キーボードショートカット

| キー | 動作 |
|------|------|
| `1`〜`28` | 出席番号で児童を選択（1,2は350ms待ち、3〜9は即確定、2桁は確定） |
| `Enter` | 保存して次の児童へ |
| `Esc` | 選択をクリア |
| `Ctrl+Z` | 直前の操作を取り消し（Undo） |
| `Ctrl+Y` または `Ctrl+Shift+Z` | やり直し（Redo） |
| `A` / `T` / `O` | 一人で / 先生と / 他クラスと |
| `F1`〜`F8` | シーン切替 |
| `Tab` | モード切替（シンプル ↔ 相手×活動） |
| `Shift+?` | ヘルプモーダル |

## AI副担任との連携

設定タブの「🧠 AI副担任向け JSON」ボタンで、`student-cards-enriched.json` 形式に統合可能なファイルを出力。
出力されるJSONには児童ごとの partners / activities / degree / asymmetry / クラスタ情報が含まれる。

## 複数PC同期

```bash
# 自宅PC: ブラウザでJSONエクスポート → デスクトップに interaction-YYYY-MM-DD.json
python C:/Users/K5610/scripts/interaction_sync.py push

# 校務PC側
python C:/Users/K5610/scripts/interaction_sync.py pull
# → デスクトップに interaction-import.json が降ってくる
# → ブラウザで「設定」→「JSONインポート」から読み込み（追加モード推奨）

# 2つのJSONを統合 (重複ID除外+timestamp新しい方優先)
python C:/Users/K5610/scripts/interaction_sync.py merge a.json b.json
```

## 学年アーカイブ（新年度切替）

新年度開始前に、設定タブ→「📦 現在のデータをアーカイブ」を実行。
- localStorageに `interactionApp_archive_YYYY_5年4組` として退避
- 同時にJSONダウンロード（バックアップ）
- 「📚 アーカイブ一覧」で過去年度のデータを確認・削除可能

その後、`students.js` の名簿を新クラスに差し替え→「全記録を削除」(直近24h以内のBU必須)で新年度開始。

## 既知の制限

- 1端末・1ブラウザの localStorage に依存。共有・同期はJSONエクスポート経由
- データが10000件を超えるとUI描画が重くなる可能性（学年末にアーカイブ推奨）
- 履歴タブは500件までしか表示しない（フィルタで絞り込み）
- ブラウザのデータ削除設定で消える可能性 → 自動BU警告バナーが7日/14日で出る

## 困ったとき

- **画面が真っ白**: F12 → Console タブでエラー確認 → 設定タブのバックアップから復元
- **localStorage破損**: バックアップキー `interactionApp_v1_backup` から復旧可能（F12コンソールで `localStorage.setItem('interactionApp_v1', localStorage.getItem('interactionApp_v1_backup'))` → リロード）
- **動作確認済ブラウザ**: Microsoft Edge 推奨 / Chrome 可 / Firefox 未確認
